##二．部署环境
为什么使用HEXO博客框架呢？
第一个是如果在搭博客过程中遇到各种各样的问题，网上解决方案比较多。有很多社区可以进行讨论。对于那些刚开始接触博客经验比较少的同学帮助很大。第二个是HEXO在各个平台，不管是Windows或者是Linux又或者是MacOS系统上操作都比较类似，命令都大差不差。

我的这个@MUSTPIGLET博客主要是模仿:https://www.codesheep.cn这篇博客来搭建的。

首先,下载并安装Node.js（下载地址：https://nodejs.org</https://nodejs.org>），记得选择LTS（长期支持版）。
接下来打开CMD命令提示符（管理员）注意从这一步开始所有指令都是在管理员模式下的命令提示符下操作，输入:node –v查看node.js 版本，确认是否安装成功。同理npm，如下图所示。

补充知识：NPM是随同Node.JS一起安装的包管理工具，能解决Node.JS代码部署上的很多问题，常见的使用场景有以下几种：
允许用户从NPM服务器下载别人编写的第三方包到本地使用。
允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

然后是下载Git（https://git-scm.com/downloads），在这里笔者在下载时候发现下载速度非常慢，只有几百b/s。如果在下载时候遇到了和我一样的情况可以从国内镜像源下载，这样速度可以提高很多，方法在这里就不详细描述了。然后按默认选项安装即可。

安装完成后，在文件夹中找到“Git”->“Git Bash”，蹦出一个类似命令行窗口的东西，就说明Git安装成功！
安装完成后，还需要最后一步设置，在git-base.exe中输入下面的命令：
$ git config —global user.name ”Your Name”
$ git config —global user.email “email@example.com”

因为Git是分布式版本控制系统，所以每个端都要自报家门。这里要注意的是 git config命令的—global参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定使用不同的用户名和Email地址。

因为HEXO需要node.js支持，在完成上面这些步骤就可以开始正式安装HEXO博客框架了。

选装cnpm
说明：因为npm这款包管理器安装插件是从国外服务器下载，受网络影响大，可能出现异常，如果npm的服务器在中国就好了，所以我们乐于分享的淘宝团队干了这事。

安装使用淘宝cnpm镜像源命令如下：
npm install -g cnpm --registry=https://registry.npm.taobao.org
-g：全局安装
可以输入cnpm检测是成功。
可以输入cnpm –v查看版本。

接下来使用安装好的cnpm全局安装hexo 输入如下命令：
cnpm install -g hexo-cli
输入hexo –v 查看版本（确认安装成功）！：

 
 
 
##三．初始化博客

首先新建一个文件夹用来存放初始化博客，地址随意，因为Git和HEXO都是全局安装的。
接下来新建博客，在命令提示符窗口进入之前新建的文件夹内输入指令：
hexo init

运行hexo初始化结果输入指令：
hexo s

在未部署时候的地址http://localhost:4000
可以在浏览器输入地址查看初始化页面。

 
 
 

##四．写博客：

新建博客文章命令：
hexo n ”文章标题”
新建完成后打开 block 文件夹下的 source 找到 _posts ,可以看到一个以.md结尾的文件，就可以开始写博客啦。在这里比较推荐使用Notepad++来编辑。
基本语法规则参见：http://www.mdeditor.com/

编辑完成后保存退出
清理释放命令：
hexo clean
生成新博客命令：
hexo g

 
 
 

##五．将博客部署到远端
在GitHub上新建一个仓库 new repository
地址为: githubID.github.io
例如： https://mustpiglet.github.io

接下来安装插件，将本地博客部署到GitHub上
cnpm install --save hexo-deployer-git

配置config.yml文件,用文件编辑器打开，找到deploy输入如下：

type: git
repo: https://github.com/mustpiglet/mustpiglet.github.io.git
branch: master

注意！删去type：后面原有的引号，在冒号后面一定要输入要空格！

部署到仓库命令：
hexo d
